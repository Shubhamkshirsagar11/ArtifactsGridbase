Dev notes
Developer notes
Documents section — behavior spec
Prototype annotation — not part of the product UI
LABEL FIELD — OPTIONAL
Display-only name shown on the agreement's Documents tab and on each covered local. Empty is valid — fall back to the filename. It never affects whether Continue is enabled.
DEFAULT STATE OF A NEW FILE
Every freshly uploaded file enters as needs tagging — amber row, "needs tagging" pill, control reads What's in this file? ▾. The system never auto-classifies a file.
TAG CONTROL
Contract / Wages / Benefits are multi-select checkboxes. Supporting document is an exclusive radio: choosing it clears the three content tags, and choosing any content tag clears Supporting. The closed control always lists the chosen tags as text — e.g. Contract · Benefits ▾ — never a count like "2 selected".
COVERAGE BAR · recomputes live
Sits directly below the list. Three tones:
GRAYlist empty — Coverage: Contract — · Wages — · Benefits — · upload the agreement paperwork to continue
AMBERany file untagged, or a content type uncovered — shows ✓/— per type plus the blocker, e.g. … · 1 file needs tagging — tell us what Scan_04-22.pdf contains to continue.
GREENall files tagged and all three types covered — … — all files tagged; ready to continue.
CONTINUE BUTTON
Enabled only when there is at least one file, no file is untagged, and Contract + Wages + Benefits are each covered by ≥1 file. Label values are not part of this check.
EMPTY LIST
The whole section collapses to one dashed dropzone: ⬆ Drag PDFs here, or click to upload with the sub-line You can drop several at once — you'll tag each file after it uploads.
UPLOAD ENTRY POINTS
"+ Add document", clicking the dropzone, or dragging files anywhere onto the list. All new files arrive as needs tagging.
