Classifications — the rate book model
Prototype annotation — not part of the product UI
WHAT A RATE BOOK IS
A rate book is one self-contained classification list inside an agreement. Some agreements hold several — e.g. Local 1002 keeps a separate list for Outside Line work and for Transmission Technician work, each with its own job classes and its own base rate that the rest are calculated from.
ONE BOOK BY DEFAULT
A new agreement starts with one rate book already created. The user must name it before adding classifications — the add row stays disabled until then. Every book (even a single one) shows a "Rate book:" header; adding a second via + Add rate book is what surfaces the grouping.
CLASSES & DERIVATION
Every class — including the first — has a derivation: Absolute $ (a base rate entered in Step 3 · Wages), % of base, + $ offset, or Same as. The base is any other class in the same book. There is no special "anchor" type — the base rate is simply a class left on Absolute $ that others point to.
NAMES ARE PER-BOOK
The same class name may appear in different books at different pay. Class names must be unique within a book, not globally.
CONTINUE GATE
Enabled only when every book is named, every book has at least one class, every class is named, and names are unique within their book.
ROW ERRORS
Name is required · Duplicate class name (within this rate book) · Select a base class · Circular derivation · Percent must be between 0 and 1000.
