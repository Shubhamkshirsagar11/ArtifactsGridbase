Developer notes
Review — behavior spec
Prototype annotation — not part of the product UI
PUBLISH GATE
Publish is disabled until every step is valid and zero classes are Unmapped (Not Available counts as handled).
OVERTIME ROW
The Overtime/Double-time row only becomes an amber "— confirm" warning when the user never touched the default multipliers on the Wages step. Here they were reviewed, so it is a green row.
BENEFIT P1 VALUE
The benefit preview's P1 value is the value in effect on period 1's first day — so a fund with its own change dates (Lineco) shows a real number, never a blank.
SUPERSEDE CHECKBOX
Checking the supersede box archives the prior agreement at the moment this one publishes, and records which agreement replaced it. Leaving it unchecked keeps both active — legitimate when a local runs two different agreements at once.
SINGLE BOOK / COLUMN
With one rate book / one wage column, the previews render without book headers and with a single value column — exactly like the current screenshot.
